## What Phase 11 is

Phase 11 is the first clean forward-architecture phase after the Phase 10 cleanup closeout.

It builds onionskin around exactly four layers:

1. pipeline-local outputs
2. shared code/modules
3. controller/run-level metadata
4. structural renaming, file renaming

This phase is where completeness and forward implementation resume. It is not allowed to rely on deprecated shared emitted-output ownership patterns.

---

## Core design rules

1. Pipeline-derived outputs belong under the pipeline that computes them.
2. Shared computation belongs in modules, not in shared emitted-analysis directories.
3. Controller/run-level artifacts must remain analysis-neutral.
4. No pipeline is the default, canonical, authoritative, or main pipeline.
5. Until explicit synthesis exists, each pipeline remains independent in both `01-prior/` and
   `02-posterior/`.
6. Cross-pipeline synthesis remains a separate explicit layer above pipeline-local products.
7. Completeness work must respect pipeline-local ownership rather than flattening differences
   into centralized intermediate files.
8. Any Phase 11 change that alters emitted paths, output schemas, or inspection-relevant
   analysis surfaces must update the affected utilities, inspectors, and regression helpers in
   the same change rather than leaving compatibility drift for later.
9. As new pipeline-local features come online, extend the relevant inspection and regression
   surfaces (for example inspectors, evaluation harnesses, and helper tests) where those
   features are now expected to be visible or testable.
10. `multistage` remains an input-data property, not the name of the growth pipeline.
11. `--pipelines all` means all three pipelines: `growth`, `per-stage`, and `hmm`.

---

## Active carry-over decisions from late Phase 10

- The long-term pipeline term remains `growth-model`, while `multistage` remains the data/property term.
- The retained grouped pipeline order/names are `01-hmm/`, `02-growth-model/`, `03-per-stage-mean-shift/`.
- The aligned future engine/module naming direction remains `hmm_engine.py`, `growth_model_engine.py`, and `per_stage_mean_shift_engine.py`, respectively.
- Until explicit cross-pipeline synthesis exists, each pipeline remains independent in prior/posterior space and should produce its own posterior manifest from its own prior outputs.
- The first synthesis target is call-level synthesis across pipeline-local prior outputs; downstream analyses can then be recomputed from the synthesized call set rather than treated as the first synthesis problem.
- The pipeline capability matrix is now written explicitly in `PIPELINE_SPEC.md` and remains code-owned by `onionskin_core/autodetect.py::decide_mode()`.

---

## Non-goals

- Do not revive deprecated shared emitted-output ownership patterns.
- Do not approximate synthesis through centralized intermediate files.
- Do not treat growth as the owner of common analysis surfaces.

## Priority 11.1 — Pipeline-local ownership architecture

**Status:** CLOSED — all four open items verified (`v0.10.47`)

### Goal

Define and enforce canonical emitted ownership for `growth`, `per-stage`, and `hmm`.

### Scope statement

Pipeline-derived outputs must emit only under the subtree of the pipeline that computes them,
and the supported pipeline-composition surface must be defined against that ownership model.

### Required outcome

- The ownership boundary for each pipeline subtree is explicit.
- Pipeline-derived intermediates and downstream outputs no longer depend on deprecated shared
  emitted-output assumptions.
- Controller/run-level metadata is clearly separated from pipeline outputs.
- The supported pipeline-composition surface is explicit for `growth`, `per-stage`, `hmm`, mixed
   subsets, and `--pipelines all`.
- Capability-based pipeline admissibility remains explicit as a function of stage count,
   replicate structure, and denominator/reference availability.
- The live planning surface includes an explicit written capability matrix rather than relying on
  code/history alone.

### Completion snapshot

- Growth-owned prior/posterior outputs now emit directly into the grouped growth subtree rather than being written flat and relocated later.
- The grouped growth subtree now owns both its root-level artifacts and its numbered secondary families directly:
  - `01-stage_median_within_calls/`
  - `02-summary_bedgraphs/`
  - `03-summits/`
  - `04-summit_refinement/`
- Controller-level grouping roots remain analysis-neutral for fresh runs (`.run_meta.json`, `00-INDEX.md`).
- Affected helper and regression surfaces were updated in the same change set, including `scripts/summit_inspector.py` and `tests/test_pipeline.py`.
- Legacy flat multistage compatibility for stale `dev/runs` / `toy_out` artifacts was intentionally retired instead of preserved.

### Implementation plan

1. Define the canonical emitted-output boundaries for each pipeline subtree.
2. Ensure pipeline-derived intermediate tracks and downstream analysis outputs are owned only by
   the emitting pipeline.
3. Define the allowed controller/run-level metadata surface separately from pipeline outputs.
4. Define the supported pipeline-composition surface against that ownership model, including the
   settled `--pipelines all` contract.
5. Write and maintain an explicit capability matrix as a function of stage count, replicate
   structure, and denominator/reference availability.
6. Remove any residual architectural dependence on deprecated shared-output ownership patterns.
7. Treat the three pipelines as peers throughout output ownership and planning language.

Extract duplicated computation into shared modules without creating shared emitted analysis

def detect_bin_size(path_or_df, max_lines: int = 5000) -> int:

    Uses the statistical mode of observed bin sizes rather than the first bin.
    Some input bedGraphs have a non-representative first bin (e.g. a different
    bin size at the very start of the file); the mode is robust against that edge
    case.  See DECISIONS.md — 2026-04-15 for the full rationale.

    if isinstance(path_or_df, pd.DataFrame):
        df = path_or_df
        sizes = (df["end"].astype(int) - df["start"].astype(int)).head(max_lines).to_numpy()
        sizes = sizes[sizes > 0]
