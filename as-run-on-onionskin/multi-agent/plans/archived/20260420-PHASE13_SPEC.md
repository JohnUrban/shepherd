## Final 14-Step Layout for `03-rcn-mean-shift/`

| Step | Dir | Contents | Implementing Priority |
|------|-----|----------|-----------------------|
| 01 | `01-stage-medians/` | Per-stage aggregate medians (base); runtime `hires-{bin}/` subdirs | 13.1 |
| 02 | `02-chrom-norm/` | Stage-level chrom-normalized tracks; `indiv_samples/` subdir | 13.1 (structure + stage-level); 13.5 (per-sample content) |
| 03 | `03-detection/` | Pre-dedup calls TSV + BED | 13.1 |
| 04 | `04-summits/` | Per-stage raw summit BEDs | 13.1 |
| 05 | `05-deduplication/` | Dedup report TSV + post-dedup canonical calls TSV + BED | 13.1 |
| 06 | `06-shape-filter/` | Shape scores TSV, collapsed repeats TSV+BED, rejected BED | 13.1 |
| 07 | `07-summit-refinement/` | Parabola-fit refined summits; hires summit BEDs | 13.3 |
| 08 | `08-multistage-unification/` | Unified cross-stage calls TSV+BED; unified collapsed repeats TSV+BED | 13.1 |
| 09 | `09-gap-analysis/` | Gap-annotated calls, gap analysis TSV, recommendation BEDs | 13.4 |
| 10 | `10-amplicon-metrics/` | Per-unified-amplicon metrics (width, peak RCN, shape score, summit) | 13.4 |
| 11 | `11-plots/` | Profile, APS, summit-fit, genome-overview plots | 13.6 |
| 12 | `12-aps/` | APS matrices, per-locus APS scores (≥1 non-ref stage) | 13.5 |
| 13 | `13-clustering/` | Cluster assignments, ordering tables (2+ stages + 3-4+ samples) | 13.5 |
| 14 | `14-notebook/` | APS exploration + gap analysis Jupyter notebooks | 13.7 |

Steps 09–14 are created by Priority 13.1 as empty reserved directories. They are populated
by their implementing Priority. This prevents renumbering when the pipeline gets meatier.

**Note on posterior output placement:** Posterior outputs do **not** live inside
`03-rcn-mean-shift/` as a numbered step. Following the established pipeline-family pattern
(matching growth and HMM), the posterior rerun lives under the top-level
`02-posterior/03-rcn-mean-shift/` grouping — the same 14-step layout but rooted under
`02-posterior/`. The posterior manifest and sidecars are written to `02-posterior/`
(in the posterior grouping dir). See Priority 13.9 for full details.

**Note on `hires-{bin}/` under `01-stage-medians/`:** These subdirs are created at
**runtime** by the engine when hires manifests are provided — not pre-created by
`ensure_layout_dirs()`. `ensure_layout_dirs()` only creates the base step dirs.

**Note on `02-chrom-norm/indiv_samples/hires-{bin}/`:** This is explicitly **not needed**.
APS reads per-sample tracks at base resolution. Hires is aggregate-only for summit
refinement. Do not create this subdir.

```python
    # --- Step 07: parabola summit refinement ---
    if not df_passing.empty:
        chrom_arr = full_df["chrom"].to_numpy()
        x_all = ((full_df["start"].to_numpy(dtype=float) + full_df["end"].to_numpy(dtype=float)) / 2.0)
        y_all = np.log2(np.maximum(np.exp2(np.where(np.isfinite(full_df["log2"].to_numpy()),
                                                     full_df["log2"].to_numpy(), 0.0)), 1e-9))

        summit_est_rows = []
        refined_peaks_para = []
        for row in df_passing.itertuples():
            chrom = str(row.chrom)
            cmask = chrom_arr == chrom
            wL = max(float(row.peak - row.start), 1.0)
            wR = max(float(row.end - row.peak), 1.0)
            resolutions = [(bin_size, x_all[cmask], y_all[cmask])]
            try:
                ps, pu = refine_summit_parabola(resolutions, float(row.peak), wL, wR)
            except Exception:
                ps, pu = float(row.peak), float("nan")
            refined_peaks_para.append(int(round(ps)))
            summit_est_rows.append({
                "chrom": chrom,
                "start": int(row.start),
                "end": int(row.end),
                "peak_raw_bp": int(row.peak),
                "parabola_summit_bp": int(round(ps)),
                "parabola_uncertainty_bp": float(pu),
                "stage": int(st),
            })

        df_summit_ests = pd.DataFrame(summit_est_rows)
        summit_est_tsv = os.path.join(
            rcn_ms_steps["summit-refinement"],
            f"{out_prefix}_stage{st}_summit_estimates.tsv",
        )
        df_summit_ests.to_csv(summit_est_tsv, sep="\t", index=False)

        tmp_refined = df_passing.copy()
        tmp_refined["peak_parabola"] = refined_peaks_para
        _calls_to_bed_summits(
            tmp_refined,
            os.path.join(
                rcn_ms_steps["summit-refinement"],
                f"{out_prefix}_stage{st}_summit_refinement.bed",
            ),
            bin_size,
            peak_col="peak_parabola",
        )
        log.verbose(f"[rcn-mean-shift] stage {st}: parabola summit refinement complete ({len(summit_est_rows)} call(s))")
```

```python
def _write_rcn_amplicon_metrics(
    unified_calls: pd.DataFrame,
    out_dir: str,
    out_prefix: str,
) -> None:
    """Write per-amplicon metrics TSV from the unified stage calls.

    Filters to final_classification == 'amplicon' rows only.
    Derives width_bp; renames peak → summit_bp and best_shape_score_raw → best_shape_score.
    """
    amp_calls = unified_calls[unified_calls.get("final_classification", pd.Series()) == "amplicon"]
    if amp_calls is None or (hasattr(amp_calls, 'empty') and amp_calls.empty):
        return
    amp_calls = amp_calls.copy()
    amp_calls["width_bp"] = amp_calls["end"] - amp_calls["start"]
    amp_calls = amp_calls.rename(columns={
        "peak": "summit_bp",
        "best_shape_score_raw": "best_shape_score",
    })
    metric_cols = [
        "chrom", "start", "end", "width_bp", "summit_bp",
        "best_shape_score", "best_stage1_score_z",
        "stages_present", "stage_state_trail",
        "support_stage_count", "final_classification",
    ]
    metric_cols = [c for c in metric_cols if c in amp_calls.columns]
    out_path = os.path.join(out_dir, f"{out_prefix}_amplicon_metrics.tsv")
    amp_calls[metric_cols].to_csv(out_path, sep="\t", index=False)
    log.verbose(
        f"[rcn-mean-shift] step 10: wrote {len(amp_calls)} amplicon metric row(s) "
        f"to {os.path.basename(out_path)}"
    )
```

### Detailed findings

**FIND-R1-1 — CRITICAL: per-sample normalization not implemented**
`02-chrom-norm/indiv_samples/` is pre-created but empty (Apr 17 timestamp). No per-sample
bedGraph writing pass exists anywhere in `rcn_mean_shift_engine.py::run_rcn_mean_shift()`.
The engine receives `sample_tracks[sid][chrom]` DataFrames where the `log2` column is already
chrom-median normalized (via `add_norm_log2()` in `refinement.py`, which computes
`norm = value / per_chrom_median` then `log2 = log2(norm)`). These normalized values just
need to be written to disk per sample.

**FIND-R1-2 — HIGH: APS not wired**
`12-aps/` is pre-created but empty. `compute_and_write_aps` is imported at `onionskin.py:46`
but is never called anywhere in the RCN mean-shift pipeline path
(`_run_rcn_mean_shift_controller()` in `onionskin.py` or the engine). No APS computation
occurs.

**FIND-R1-3 — HIGH (spec error): `13-clustering/` target state is wrong**
`compute_and_write_aps()` writes ALL outputs — APS tables, matrices, clustering assignments,
and order tables — to a single `out_dir`. The actual clustering output filenames written to
`12-aps/` are: `aps_clusters_raw.tsv`, `aps_clusters_zscore.tsv`,
`aps_dendrogram_order_raw.tsv`, `aps_dendrogram_order_zscore.tsv`, `aps_order_area.tsv`,
`aps_order_summit.tsv`. There is no separate clustering step and nothing will ever write
to `13-clustering/`. The spec's target state for `13-clustering/` — with
`{prefix}_cluster_assignments.tsv` and `{prefix}_ordering.tsv` — is wrong on both
location and filename. The correct validation target is files inside `12-aps/`, following
the growth pipeline convention.

**FIND-R1-4 — HIGH: architectural constraint on `compute_and_write_aps` call site**
`compute_and_write_aps(manifest_path: str, ...)` always takes a file path to the raw manifest
and applies chrom-median normalization internally via `compute_sample_rcn_tracks(manifest_path,
norm_mode='chrom-median', ...)`. There is no `norm_mode='none'` passthrough. The RCN engine
(`run_rcn_mean_shift()`) does not have access to the manifest path — it receives pre-loaded
DataFrames. Solution: write per-sample bedGraphs from in-memory data in the engine
(FIND-R1-1), and call `compute_and_write_aps` from the controller
(`_run_rcn_mean_shift_controller()`) using `args.manifest`. The controller has `args`
available and therefore has the original manifest path. The `indiv_samples/` bedGraphs and
the APS normalization are independently computed using the same chrom-median formula —
mathematically identical, no double-normalization.

**FIND-R1-5 — HIGH: unified BED call_id format incompatible with `parse_loci_bed()`**
The existing unified BED (`08-multistage-unification/{prefix}_unified_stage_calls.bed`)
is written by `_calls_to_bed_domains_classified()` with name column format
`{chrom}:{start}-{end}|{final_classification}|{stage_state_trail}` (6-column BED).
`parse_loci_bed()` in `aps.py` uses column 4 verbatim as `call_id`, so it would produce
call_ids like `II:800000-1500000|amplicon|1_->2_->3_`. These break APS summit-lookup
(summit_bp_map uses clean `{chrom}:{start}-{end}` keys) and downstream label matching.
Fix: the engine must additionally write a clean 4-column APS BED alongside the existing
unified BED, with `call_id = {chrom}:{start}-{end}` and no extra fields.

**FIND-R1-6 — HIGH: HMM step 14 called unconditionally**
`run_step14_hmm_aps()` is called at `hmm_engine.py:1238` with no gating on stage count
or non-reference stage presence. The spec requires gating at ≥1 non-reference stage.
The HMM engine has a `groups` variable (list of stage groups from the manifest) and
`step14_ref_stage` parameter. A guard of
`if len([g for g in groups if int(g) != int(step14_ref_stage)]) >= 1:` would implement
the correct gating before the `run_step14_hmm_aps(...)` call.

**FIND-R1-7 — INFO: HMM step 16 is trajectory clustering, not APS clustering**
`run_step16_hmm_clustering()` reads from `fork_travel_metrics.tsv` (step 12 fork-travel
output) and clusters HMM trajectories by fork-travel feature vectors. This is a completely
different concept from APS-based sample clustering. The spec's ≥2+3-4 gating rule is for
APS sample clustering (which is embedded inside `compute_and_write_aps()`), not for HMM
trajectory clustering. No gating change is needed for step 16. The call at
`hmm_engine.py:1257` is correct as-is — if `fork_travel_metrics.tsv` is missing the function
raises RuntimeError, which is caught by the surrounding try/except.

```python
    result: Dict[str, Tuple[Optional[int], str]] = {}
    paths = sorted(glob.glob(os.path.join(samples_dir, '*.RCN.bedGraph')))
    if not paths:
```

### Loose End A — `onionskin_core/rcn_mean_shift_engine.py` lives at the wrong level

#### Problem

The RCN mean-shift batch dispatch engine (`onionskin_core/rcn_mean_shift_engine.py`, 607 lines)
lives at the **top level of `onionskin_core/`**, while the two peer batch engines — the HMM and
the growth model — both live under **`onionskin_core/engines/`**:

```
onionskin_core/engines/growth_model_engine.py   ← batch engine
onionskin_core/engines/hmm_engine.py            ← batch engine
onionskin_core/rcn_mean_shift_engine.py         ← batch engine — WRONG LEVEL
```

The top level of `onionskin_core/` is the home for utility / helper modules (`detection.py`,
`gap_analysis.py`, `timing.py`, `io.py`, etc.). Batch engines belong in `engines/`.

This inconsistency was introduced during Phase 13 Priority 13.1: the spec planned a
`git mv onionskin_core/per_stage_mean_shift_engine.py onionskin_core/rcn_mean_shift_engine.py`
rename-in-place, not a move into `engines/`. Copilot then deviated by creating a new file
instead of a `git mv`, and the architectural peer-placement question was not raised. The
Priority 13.1 auditors accepted the result as "functionally equivalent."

- **Role:** batch multi-stage dispatcher; receives `sample_tracks`, `sample_ids`, `stages`,
  `rcn_ms_steps` dict, and `out_prefix`; orchestrates the full 14-step RCN mean-shift pipeline
  including per-stage calling, unification, gap analysis, APS, plots, and notebooks.
- **Public API exported:**
  - `run_rcn_mean_shift()` — primary batch entry point (line 406)
  - `run_single_stage()` — single-stage calling helper (re-exported from
    `rcn_mean_shift_helpers.py`)
  - `apply_shape_filter_tsv()` — post-hoc shape filter over a saved TSV (re-exported from
    `rcn_mean_shift_helpers.py`)
  - `apply_peak_proximity_dedup_tsv()` — post-hoc dedup over a saved TSV (re-exported from
    `rcn_mean_shift_helpers.py`)

### Summary

Completed the Role 3 Final Overseer wrap-up audit of Phase 13. While the per-stage restructuring
and name migration to `rcn-mean-shift` were successfully applied and validated for the primary
pipelines, a deep audit of the `--posterior` integration revealed four cross-cutting logic bugs
that break standalone `rcn-mean-shift` posterior runs. These regressions are currently masked
by the default `--pipelines all` behavior in existing regression tests.

### Detailed Findings

1.  **FIND-R3-1: Early return in standalone `rcn-mean-shift` block.**
    In `onionskin.py`, the block handling `dec.mode == "multistage" and _pipelines_include(args, "rcn-mean-shift")`
    returns immediately after running the controller, skipping the `_run_posterior()` block at the end of `main`.

2.  **FIND-R3-2: Hardcoded APS directory source.**
    `_run_posterior()` and `_reconstruct_prior_layout()` hardcode the APS prerequisite directory
    to `02-growth-model/13-aps/`. Standalone `rcn-mean-shift` runs correctly emit APS to
    `03-rcn-mean-shift/12-aps/`, so posterior runs on these priors will fail to find cluster assignments.
    Confirmed via `output_layout.py` that the keys `"aps_dir"` (growth) and `"rcn_aps_dir"` (RCN)
    are distinct but only the former is currently checked.

3.  **FIND-R3-3: Unconditional growth-model execution in posterior.**
    `_run_posterior()` unconditionally runs the `growth-model` (multistage) engine even if it
    wasn't requested, leading to empty `02-growth-model/` directories and unnecessary computation
    when only `rcn-mean-shift` or `hmm` are active.

4.  **FIND-R3-4: Regression test masking.**
    `tests/test_pipeline.py::test_posterior_rcn_mean_shift_output_exists` passes only because it
    uses default pipelines (both growth and rcn-mean-shift), satisfying the hardcoded
    `02-growth-model/13-aps/` requirement. Standalone runs currently fail this test.
